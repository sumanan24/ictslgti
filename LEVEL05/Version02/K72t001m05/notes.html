Introduction: Programming Concepts, Generations & Translators

Intro Programming Concept, Generations of Languages, and Translators

What programming is, how programming languages evolved (generations), and how source code is turned into something the computer can run (translators).

What is Programming?

Programming is the process of writing instructions (code) that a computer can understand and execute. A program is a set of instructions that tells the computer what to do step by step. Programming involves problem-solving, designing algorithms, and writing code in a programming language.

You write instructions in a language like Java or Python. The computer cannot run your words directly—something must translate your code into the machine’s own language (binary). That “something” is a translator (compiler or interpreter).

Generations of Programming Languages

Programming languages are often grouped into generations based on how close they are to the hardware and how easy they are for humans to read and write.

  • First Generation (1GL) Machine Language: Binary code (0s and 1s) that the CPU executes directly. Very fast but very hard for humans to write or read. Example: 10110010 00000001.
  • Second Generation (2GL) Assembly Language: Uses short mnemonics (e.g. ADD, MOV, JMP) that map almost one-to-one to machine instructions. An assembler translates assembly into machine code. Still tied to a specific processor.
  • Third Generation (3GL) High-Level Languages: Readable, portable languages. One statement can represent many machine instructions. Examples: C, Java, Python, C++. These need a compiler or interpreter to run.
  • Fourth Generation (4GL) Very High-Level: Languages and tools focused on specific tasks (databases, reports, forms). Examples: SQL, report generators. Less code for the same result.
  • Fifth Generation (5GL) Constraint / Natural Language: Problem-oriented; programmer may describe the problem rather than the exact steps. Used in AI and some expert systems.

1GLMachine languageBinary (0, 1)None (direct execution)

Translators: Compilers, Interpreters, and Assemblers

A translator converts program code written in one form into another form that can be executed (or into something closer to execution).

  • Assembler: Translates assembly language into machine code. One assembly instruction usually becomes one machine instruction. Used for 2GL.
  • Compiler: Translates the entire source code (e.g. C, Java) into object code or machine code (or an intermediate form like bytecode) before execution. The program is then run separately. Fast execution; need to recompile when source changes. Examples: GCC (C), javac (Java → bytecode).
  • Interpreter: Translates and executes source code line by line (or chunk by chunk). No separate executable file; the interpreter runs your code directly. Easier to test and change, but often slower than compiled code. Examples: Python interpreter, JavaScript in the browser.
TranslatorInputOutputWhen translation happens
AssemblerAssembly codeMachine codeBefore execution (one pass)
CompilerHigh-level source (e.g. C, Java)Object/bytecode/machine codeBefore execution (whole program)
InterpreterHigh-level source (e.g. Python)ExecutionDuring execution (line by line)

An assembler turns assembly into machine code. A compiler turns your whole program into something the machine (or VM) can run, then you run it. An interpreter reads your code and does what it says as it goes—no separate “compiled file” to run. Java uses both: javac compiles to bytecode; the JVM interprets (or JIT-compiles) the bytecode. Python is usually interpreted.

Chapter 1: Basics of Computer Programming

1 Algorithms

What an algorithm is, properties of a good algorithm, and how to write simple algorithms.

What is an Algorithm?

An algorithm is a step-by-step procedure to solve a problem or perform a task. It is written in plain language (or pseudocode) and is independent of any programming language.

An algorithm is like a recipe. You have clear steps: first do this, then do that. If you follow the steps correctly, you get the correct result. It must be finite (ends after some steps), unambiguous (each step has one meaning), and effective (each step can be done).

Properties of a Good Algorithm

  • Input: Zero or more well-defined inputs.
  • Output: At least one well-defined output.
  • Finiteness: Must terminate after a finite number of steps.
  • Definiteness: Each step is clear and unambiguous.
  • Effectiveness: Each step is doable in a finite time.

Example: Algorithm to Find the Larger of Two Numbers

1. START
2. Read number A
3. Read number B
4. IF A > B THEN
      Display A
   ELSE
      Display B
   END IF
5. STOP

2 Flowcharts

Flowchart symbols and how to represent algorithms visually.

What is a Flowchart?

A flowchart is a diagram that shows the flow of control and steps in an algorithm using standard shapes. It makes logic easy to see and discuss.

Common Flowchart Symbols

Oval: Start / Stop (terminator)

Rectangle: Process

Parallelogram: Input / Output

Diamond: Decision

Arrows: Flow of control

  • Oval: Start / Stop (terminator)
  • Rectangle: Process (e.g. calculation, assignment)
  • Parallelogram: Input / Output
  • Diamond: Decision (Yes/No branch)
  • Arrows: Flow of control

Start and end are ovals. A normal step (like "add 1 to count") is a rectangle. When you need to choose (e.g. "Is count > 10?") you use a diamond with two arrows: one for Yes and one for No.

Example: Flowchart for "Display larger of A and B"

Start → Input A, B → Decision: Is A > B? → Yes: Output A → No: Output B → End.

Questions
  1. Which shape is used for a decision?
  2. Draw a flowchart for: Read a number; if it is even, print "Even", else print "Odd".

3 Pseudocode

Writing algorithms in pseudocode—readable, language-independent steps.

What is Pseudocode?

Pseudocode is a way of writing an algorithm using a mix of natural language and programming-like keywords (IF, ELSE, WHILE, FOR). It is not real code but is easy to translate into Java, Python, or any language.

Common Pseudocode Keywords

  • INPUT / READ get data
  • OUTPUT / PRINT / DISPLAY show result
  • IF ... THEN ... ELSE ... END IF selection
  • WHILE ... DO ... END WHILE loop (pre-test)
  • FOR ... TO ... END FOR count-controlled loop

Example: Pseudocode for Sum of First N Numbers

INPUT N
SET sum = 0
SET i = 1
WHILE i <= N DO
   sum = sum + i
   i = i + 1
END WHILE
OUTPUT sum

Chapter 2: Introduction to Object-Oriented Programming

4 Procedural vs Object-Oriented Programming

How procedural programming differs from OOP and why OOP is used for large systems.

Procedural Programming

Programs are organized around functions/procedures that operate on data. Data and functions are separate. Data is often global or passed as parameters. Examples: C, Pascal, early style in many languages.

Object-Oriented Programming (OOP)

Programs are organized around objects. Each object has data (attributes) and behaviour (methods) bundled together. Emphasis on classes, encapsulation, inheritance, and polymorphism. Examples: Java, Python, C++, C#.

AspectProceduralObject-Oriented
Unit of designFunction / procedureClass / Object
Data & behaviourSeparateEncapsulated in object
ReuseFunctions, librariesInheritance, composition
Data accessOften global or parametersThrough object's methods (controlled)

In procedural style you write functions and pass data to them. In OOP you create "things" (objects) that have both data and actions; the object knows how to use its own data.

Chapter 3: Classes and Objects

5 Classes and Objects Real-world Examples

Class = blueprint; Object = instance. Real-world analogy.

Class

A class is a blueprint or template that defines the attributes (data) and methods (behaviour) that objects of that type will have. It does not hold actual data until you create an object.

Object

An object is a concrete instance of a class. It has its own copy of the attributes and can call the methods. Example: Class = Car; objects = myRedCar, yourBlueCar.

Real-world: Class = "Student" (has name, id, grade). Objects = "Nimal" (Nimal, 001, 10), "Kamala" (Kamala, 002, 10). Each student is one object with their own name, id, and grade.

Variables and Data Types

Attributes are stored in variables. Each variable has a data type that defines what values can be stored and what operations are allowed.

PurposeJavaPython
Integerintint
Decimaldouble, floatfloat
TextStringstr
True/Falsebooleanbool
Charactercharstr (length 1)

Attributes and Methods

  • Attributes (fields/properties): data that each object holds (e.g. name, age).
  • Methods: functions that belong to the class; they operate on the object's data (e.g. getName(), setAge()).

Complete Example: Student Class

Java
public class Student {
  private String name;
  private int id;
  public Student(String n, int i) {
    name = n; id = i;
  }
  public String getName() {
    return name;
  }
  public void setName(String n) {
    name = n;
  }
}
Python
class Student:
  def __init__(self, n, i):
    self.name = n
    self.id = i
  def get_name(self):
    return self.name
  def set_name(self, n):
    self.name = n

6 Instantiating Objects, Accessing Attributes and Methods

How to create objects and call their methods in Java and Python.

Instantiating Objects

Instantiation = creating an object from a class. In Java: ClassName obj = new ClassName(); In Python: obj = ClassName().

Accessing Attributes and Methods

Use the object reference and the dot operator: obj.attribute, obj.method(). In Java, if attributes are private, you use getters/setters instead of direct access.

Java
// Create object
Student s = new Student();
// Access (if public) or use getters/setters
s.setName("Nimal");
String n = s.getName();
Python
# Create object
s = Student()
# Access attributes / methods
s.name = "Nimal"
n = s.get_name()

7 Constructors and Destructors

Constructor = initialize object; Destructor = cleanup (Python: __del__; Java: no direct destructor, GC handles it).

Constructors

A constructor is a special method that runs when an object is created. It is used to set initial values for attributes. In Java: same name as class, no return type. In Python: __init__(self, ...).

Java
public class Student {
  String name;
  int id;
  public Student(String n, int i) {
    name = n;
    id = i;
  }
}
// Use: Student s = new Student("Nimal", 1);
Python
class Student:
  def __init__(self, n, i):
    self.name = n
    self.id = i
# Use: s = Student("Nimal", 1)

Destructors

Java: No destructors. The JVM uses garbage collection to free memory when objects are no longer used. Python: __del__(self) is called when the object is about to be destroyed (rarely need to define it).

8 Method Overloading

Same method name, different parameter lists. Java: supported. Python: no true overloading; use default args or different names.

Concept

Method overloading = defining multiple methods with the same name but different number or types of parameters. The compiler/interpreter chooses which one to call based on the arguments.

Java
public void print(int x) {
  System.out.println(x);
}
public void print(String s) {
  System.out.println(s);
}
public void print(int a, int b) {
  System.out.println(a + b);
}
Python
def print_val(self, x, y=None):
  if y is None:
    print(x)
  else:
    print(x + y)
# Or use *args / default parameters

In Java you can have add(int, int) and add(double, double); the right one is picked by the argument types. In Python you often use one method with optional parameters.

Chapter 4: Memory Management in OOP

9 Dynamic Memory Allocation

Objects are created on the heap; references point to them.

Concept

When you create an object with new (Java) or by calling the class (Python), memory is dynamically allocated on the heap. A variable holds a reference (address) to that memory, not the object itself.

Creating an object reserves a block of memory for its data. Your variable is like a pointer to that block. You don't manually free it in Java or Python—the runtime does it.

Garbage Collection and Object Lifecycle

Garbage collection (GC) = automatic process that finds objects that are no longer reachable and frees their memory. Object lifecycle: Create (constructor) → Use → Unreferenced → Eligible for GC → Collected.

In Java: JVM runs GC. In Python: reference counting + cyclic GC. You generally do not need to write destructors for normal cleanup; use try-finally or try-with-resources (Java) / context managers (Python) for releasing resources like files.

Chapter 5: Modifiers in OOP

10 Access Modifiers and Non-access Modifiers

Control visibility (access) and behaviour (static, final, etc.).

Access Modifiers

They control who can see a class, method, or field.

  • public accessible from anywhere.
  • private only inside the same class.
  • protected same class + subclasses (and same package in Java).
  • default (package-private, Java) same package only.

Python: No strict access modifiers. Convention: _name = "protected", __name = name mangling (strong hint of "private").

Non-access Modifiers

They change behaviour, not visibility.

  • static (Java) belongs to the class, not to each object. One copy shared by all. Python: use class variables and @staticmethod / @classmethod.
  • final (Java) constant (variable) or cannot override (method) or cannot extend (class). Python: no final; use naming convention or documentation.
Java
public class Example {
  private int x;
  public static int count = 0;
  public final int MAX = 100;
}
Python
class Example:
  def __init__(self):
    self._x = 0      # convention: "protected"
    self.__secret = 0 # name mangling
  count = 0          # class variable
  MAX = 100          # class constant

Chapter 6: Encapsulation

11 Encapsulation Concept, Getters and Setters

Hide data, expose through methods; getters and setters.

Encapsulation

Encapsulation means bundling data (attributes) and the methods that operate on that data inside a class, and hiding internal details by making fields private (or protected). Access is through public methods (getters/setters) so you can validate and control changes.

Don't let outside code touch your object's data directly. Provide getName() to read and setAge(int age) to write (and inside setAge you can check e.g. age > 0).

Getters and Setters

Getter = method that returns the value of a private field. Setter = method that sets the value, often with validation. Naming: getX() / setX() (Java), or get_x() / set_x() or properties (Python).

Java
private String name;
public String getName() {
  return name;
}
public void setName(String name) {
  if (name != null) this.name = name;
}
Python
def __init__(self):
  self.__name = ""
def get_name(self): return self.__name
def set_name(self, name):
  if name: self.__name = name
# Or: @property and @name.setter

Chapter 7: Inheritance

12 Inheritance Concept, Hierarchies, Single and Multiple, Overriding

Subclass gets superclass members; single vs multiple inheritance; overriding methods.

Inheritance Concept

Inheritance = a new class (subclass/child) is based on an existing class (superclass/parent) and inherits its attributes and methods. This supports code reuse and "is-a" relationships (e.g. Dog is an Animal).

Hierarchies and Class Relationships

Class hierarchy = tree of classes: Animal → Mammal → Dog. Child can add new methods/fields and override parent methods.

Single vs Multiple Inheritance

  • Single: A class has one parent. Java: only single (class can extend one class). Python: single for the normal case.
  • Multiple: A class has more than one parent. Java: not for classes; use interfaces for multiple "types". Python: allowed (e.g. class C(A, B):).

Method Overriding

When a subclass defines a method with the same signature as the parent, the subclass version is used when called on the subclass object. In Java: @Override. In Python: just define the method again in the child.

Java
class Animal {
  void speak() { System.out.println("..."); }
}
class Dog extends Animal {
  @Override
  void speak() { System.out.println("Woof"); }
}
Python
class Animal:
  def speak(self): print("...")
class Dog(Animal):
  def speak(self): print("Woof")

Chapter 8: Polymorphism

13 Polymorphism Concept

One interface or reference, many forms; runtime behaviour depends on actual object type.

What is Polymorphism?

Polymorphism ("many forms") means the same operation or method name can behave differently depending on the actual object. Typically achieved by method overriding: a superclass reference can point to a subclass object, and the overridden method of the subclass is called.

You have a variable of type Animal. It can hold a Dog or a Cat. When you call speak(), if it's a Dog you get "Woof", if it's a Cat you get "Meow". Same call, different behaviour.

Java
Animal a = new Dog();
a.speak();  // Woof (Dog's method)
a = new Cat();
a.speak();  // Meow
Python
a = Dog()
a.speak()   # Woof
a = Cat()
a.speak()   # Meow

Chapter 9: Abstraction

14 Abstraction, Abstract Classes, Interfaces, Real-world Examples

Hide implementation details; abstract classes and interfaces.

Abstraction Concept

Abstraction means showing only what is necessary and hiding how it is done. The user sees "what" (e.g. start engine) not "how" (fuel injection, spark). In OOP: abstract classes and interfaces define contracts; concrete classes implement them.

Abstract Classes

An abstract class cannot be instantiated. It may have abstract methods (no body) that subclasses must implement, and can also have concrete methods. Java: abstract class X { abstract void m(); }. Python: inherit from ABC and use @abstractmethod.

Interfaces

An interface defines a contract (method signatures) with no implementation. Java: interface I { void m(); } class implements interface. Python: no built-in interface; use abstract base classes (ABC) with only abstract methods, or protocols.

Java
interface Drawable {
  void draw();
}
abstract class Shape implements Drawable {
  abstract double area();
}
Python
from abc import ABC, abstractmethod
class Drawable(ABC):
  @abstractmethod
  def draw(self): pass
class Shape(Drawable, ABC):
  @abstractmethod
  def area(self): pass

Real-world: Encapsulation = hide data (e.g. balance in BankAccount, access via deposit/withdraw). Abstraction = hide complexity (e.g. "drive" hides engine, transmission; "save" in an app hides file format and disk writes).

Chapter 10: Relationships Between Classes

15 Association, Aggregation, Composition

How classes relate: association (uses), aggregation (has-a, independent), composition (has-a, owns and lifecycle-dependent).

Association

Association = a general "uses" or "works with" relationship between two classes. One class has a reference to another (e.g. Student uses Library, Doctor treats Patient). They can exist independently.

Aggregation

Aggregation = "has-a" relationship where the whole contains parts, but the parts can exist without the whole (e.g. Department has Teachers; if the department is removed, teachers still exist). Weak ownership.

Composition

Composition = stronger "has-a": the whole owns the parts and is responsible for their lifecycle. When the whole is destroyed, the parts are destroyed (e.g. Car has Engine; when Car is destroyed, Engine goes with it). Strong ownership.

RelationshipMeaningLifecycle
AssociationUses / links toIndependent
AggregationHas-a (whole–part)Parts can outlive whole

Java – Composition
class Car {
  private Engine engine;
  Car() { engine = new Engine(); }

// Engine created with Car, dies with Car
Python – Aggregation
class Department:
  def __init__(self, teachers):
    self.teachers = teachers  # refs to existing
